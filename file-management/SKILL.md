# File Management Skill

## Overview
This skill provides systematic approaches to organize files into structured folders and enhance system navigation capabilities for efficient file management.

## Core Objectives

### 1. Structured File Organization
- Create hierarchical folder systems for easy access and retrieval
- Implement consistent naming conventions across projects
- Maintain clean directory structures for long-term sustainability

### 2. System Navigation Enhancement
- Master efficient file location and retrieval
- Utilize search tools effectively across the filesystem
- Develop systematic approaches to file management

## Key Procedures

### File Organization Workflow
1. **Assessment Phase**
   - Survey current file structure
   - Identify patterns and inconsistencies
   - Determine organizational hierarchy needs

2. **Structure Planning**
   - Create folder hierarchy based on project type, date, or category
   - Establish consistent naming conventions
   - Plan archive and temporary folder locations

3. **Implementation Phase**
   - Create directory structure
   - Move files to appropriate locations
   - Remove unnecessary duplicates
   - Update references where needed

### System Navigation Techniques
1. **Search Strategies**
   - Use grep/search tools for file content location
   - Leverage file listing commands efficiently
   - Utilize pattern matching for file discovery

2. **Navigation Skills**
   - Maintain awareness of current working directory
   - Use relative vs absolute paths appropriately
   - Implement efficient file path conventions

## Practical Tips

### Organization Tips
- Group related files together
- Use descriptive folder names
- Implement version control for critical files
- Create backup systems for important documents

### Navigation Tips
- Use tabs and split views for efficient comparison
- Leverage search with filters and wildcards
- Maintain organized file history
- Use shortcuts for frequently accessed locations

## Common Pitfalls to Avoid

### Organization Pitfalls
- Inconsistent naming across similar files
- Overly complex folder structures
- Not archiving old or completed projects
- Fragmenting related content

### Navigation Pitfalls
- Not utilizing search capabilities effectively
- Getting lost in deep directory structures
- Losing track of file locations
- Not maintaining file search history

## Verification Steps

### After Organization
- Verify all files are accessible
- Check that folder hierarchy is logical
- Ensure naming conventions are consistent
- Confirm search functionality works properly

### Navigation Efficiency
- Test file search capabilities
- Verify path traversal works smoothly
- Check shortcut functionality
- Ensure file relationships are maintained

## Tool Integration

### Primary Tools
- `search_files`: For finding files by name or content
- `mcp_filesystem_list_directory`: For directory browsing
- `read_file`: For file content verification
- `patch`: For targeted file edits

### Workflow Integration
- Use search tools for file discovery
- Implement batch operations for mass organization
- Leverage read/write tools for file verification
- Apply patch tool for targeted modifications

## Success Metrics

### Organizational Success
- Files are logically grouped by content type
- Naming conventions are consistent across projects
- Search operations return relevant results quickly
- Directory structures are easy to navigate

### Navigation Success
- File locations are easy to determine
- Search tools are used effectively
- Path operations work without errors
- File relationships are maintainable
