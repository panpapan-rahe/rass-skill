# Backend Reset Protocol

## Purpose
Document the new backend learning model after the reset-from-zero session.

## Active Learning Loop (Same as Frontend)
1. Pick one micro-topic for the day.
2. Build a small lab artifact.
3. Apply the concept to a real project when possible.
4. Audit the result and note the next step.

## Cadence
- Daily cron runs at 22:30.
- A missed weekday can be replaced with a manual trigger for that day's topic.
- Fase besar tetap dipakai sebagai kerangka, tapi yang mengerjakan harian adalah micro-topic kecil.

## Current Backend Level Map (4 Levels)
- Level 1 — Fundamental: API Architect, Business Logic, Database Architect, Validation
- Level 2 — Keamanan & Data: Authentication, Authorization, Security, File Storage
- Level 3 — Skalabilitas: Queue, Background Worker, Caching, Performance
- Level 4 — Engineering: Testing, Logging, Monitoring, Architecture, DevOps Integration, Backend Reviewer

## Audit Rule
Impact should be visible from week one; do not wait for a full 4-week cycle before checking progress.
