# Frontend Reset Protocol

## Purpose
Document the new frontend learning model after the reset-from-zero session.

## Active Learning Loop
1. Pick one micro-topic for the day.
2. Build a small lab artifact.
3. Apply the concept to a real project when possible.
4. Audit the result and note the next step.

## Cadence
- Daily cron runs at 22:00.
- A missed weekday can be replaced with a manual trigger for that day's topic.
- Fase besar tetap dipakai sebagai kerangka, tapi yang dikerjakan harian adalah micro-topic kecil.

## Current Frontend Phase Map
- Phase 1: The Bedrock — HTML / CSS / JavaScript fundamentals
- Phase 2: Designer's Eye — UI / UX / A11y / Design System
- Phase 3: The Engineer — Components / React / API integration
- Phase 4: The Polisher — review / performance / mentoring

## Audit Rule
Impact should be visible from week one; do not wait for a full 4-week cycle before checking progress.
